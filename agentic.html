Agentic AI for Developers

There's a gap between "I've tried a few prompts" and "AI handles a real piece of my workflow." The gap is real, and it has a map. Four sessions, small cohort, real code — from AI-augmented toolchain to a working multi-agent system you built and understand.

Who it's for

You're already using AI. It's not helping enough.

This course is for developers who've gone past "hello world" with AI tools but haven't crossed the line into genuinely useful.

The plateau

"I do use AI. It's just not helping."

You use autocomplete. You've asked Claude or ChatGPT to explain code or fix a bug. It mostly works. But it's not a teammate — it's a slightly smarter search engine. You can feel the gap between where you are and where this technology is actually going, and you don't have a clear next step.

The anxiety

The field is shifting faster than you can keep up

Every week there's a new model, a new framework, a new claim that agents will replace your job. You're not sure which parts to pay attention to. This course isn't about keeping up with announcements — it's about building one thing, end to end, that actually works. That's what makes the noise stop.

Curriculum

Four sessions. One working system.

Each session builds on the last. You leave Session 4 with a multi-agent stack running in production, plus reusable patterns for every agent you build after.

Prerequisites
  • Comfortable with Python or TypeScript
  • Basic REST API / async familiarity
  • Git basics — clone, commit, push
  • Terminal not intimidating
What to bring
  • GitHub account (free)
  • Anthropic API key (free tier covers Sessions 1–2)
  • VS Code or preferred editor
  • Node 18+ or Python 3.10+
1

The AI-Augmented Dev Toolchain

Claude Code vs. Copilot vs. Codex — what they actually do, where they differ, and how to configure them for real work. You'll set up Claude Code, connect it to a repo, and write a CLAUDE.md that tells it how your project actually works.

Model families Context files Agentic vs. autocomplete Token budgets Security posture
You leave with
  • Claude Code connected to a real repo
  • A working CLAUDE.md for your project
  • A model-selection cheat sheet
2

Build Your First Agent

Architecture patterns, tool-calling, memory, and building a real personal assistant agent from scratch. You'll wire up email scanning, calendar awareness, and persistent memory across runs — the full loop of a working agent, not a demo.

ReAct loops Tool schema design Short vs. long-term memory Structured output Agent observability
You leave with
  • A working personal assistant agent
  • Reusable tool-calling scaffolding
  • Persistent memory store (file or SQLite)
3

Multi-Agent Teams

Orchestration patterns, agent roles, and secure hosting — either AWS EC2 with Terraform or local with Tailscale for zero-cost secure access. You'll wire your Session 2 agent in as a subagent inside a hardened orchestration harness and test handoffs between agents.

Orchestrator / subagent patterns Docker Compose Least-privilege IAM Tailscale mesh networking Audit trails
You leave with
  • A running agent harness on your chosen host
  • Your agent wired in as a subagent
  • Terraform config for your infra
4

Build It — Three Apps and Your Own

Three practical applications built live through your agent stack: a daily briefing agent on a cron job, a code review agent connected to GitHub, and a research and RAG agent with citations. Then you bring an app idea of your own and we architect it together.

Cron scheduling GitHub API integration RAG: chunking + retrieval Vibe-coding workflow Production hardening
You leave with
  • 3 working deployed apps
  • Stub for your own app idea
  • Full agent stack in production
Outcomes

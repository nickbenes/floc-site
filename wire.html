Who it's for

Two groups hit the same walls

The constraint is the same — Zone B's locked-down network. The path in is different.

Outsiders new to DoD

You came in expecting your tools to work

Docker pull. npm install. GitHub Actions. An AI coding agent you can wire into VS Code. None of it works in Zone B. You've spent the first weeks figuring out why, with no map and no one to ask who's been through it.

DoD veterans hit by the mandate

You've worked Zone B for years. Now leadership says use AI.

The GenAI.mil rollout didn't come with a developer path — just a browser tab and an email that said "use it." You can copy and paste in a chat window. You can't wire it into your terminal, your repo, or anything that actually resembles how you work.

Curriculum

Four walls. Four ways through.

Each module covers one real constraint — not the theoretical version, the version you hit on a Tuesday when you need to ship something and Zone B says no.

1
Wall One

The Dependency Wall

npm install hangs. pip install times out. The package you need has 47 transitive dependencies, none of them cached. This module covers local registries, vendored packages, and a pre-staging workflow that survives an air gap without reinventing your entire stack on the other side.

2
Wall Two

The AI Mandate Gap

Leadership mandated GenAI.mil. There's no CLI, no VS Code integration, no approved path for developers specifically. This module covers what the approved tools actually support for code, how to bridge the gap between a browser-based AI and your working environment, and what to do when the approved tool simply can't reach your problem.

3
Wall Three

The Context Gap

You can generate code. Getting it from LLM output into a version-controlled, reviewed, and deployed state is a different problem — one no YouTube tutorial covers because it assumes you have internet. This module covers the flatten-and-expand pattern and a repeatable transfer workflow that doesn't depend on copy-paste heroics.

4
Wall Four

The Repeatable Workflow

Everything above combined into a daily practice. A Zone-B-native development environment: CI without GitHub Actions, code review without automated gates, a way to onboard a new developer without handing them a month of trial and error. A system that doesn't break when the tools or the mandates change — because they will.
